## 2. 檔案結構與命名

```
Learning/
  index.html                  ← 課程首頁（目錄＋進度）
  chapters/
    ch{單元}-{章}.html         ← 例：ch1-3.html、ch2-1.html
  enterprise_application_consultant_course_spec.md
  course_production_guide.md  ← 本文件
```

每新完成一章，要同步改 index.html 該章那一列：
把 `<span class="soon">製作中</span>` 換成 `<a class="golink" href="chapters/ch{n}-{m}.html">開始閱讀</a>`。

**用詞注意（使用者 2026-07-17 指定）**：
- ⚠️ 寫「之後 OO 會教」這類前向引用時，也要用「單元」，不要手滑打成「模組」。1-3～1-7 這個錯漏犯過（已修），每批完成後務必 `grep -rn 模組` 全部章節檔案確認乾淨
- UI 與所有內文一律稱「**單元**」，不用「模組」（spec 原文寫模組，產出時要轉換）
- 卡片標籤用「**情境例子** 1/2、2/2」，不用「白話解釋」
- index.html 目錄的章號欄（td.no）已設 `white-space:nowrap`，勿移除
- index.html 左側有「章節快捷索引」sidenav，由 JS 從目錄表格**自動產生**，樣式為 **Accordion**（`<details class="sn-acc">`）：每個單元一個可收合項，預設收合，展開後有「跳到目錄這個單元」連結＋各章的**條列清單**（章號＋標題、單行截斷、左側直線縮排；已製作的章直接連到章節檔，未製作的跳到目錄該列；完成的顯示綠色加 ✓）。新章只要加/改表格列即可，sidenav 不用手動維護；視窗寬度 <1200px 時自動隱藏

## 6. 語氣與文字規則（使用者明確要求）

1. **溫柔的老師語氣**：像有耐心的老師對非資訊本科的學生說話。可以用「別擔心」「慢慢來」「沒關係」「準備好了嗎」。答錯要安撫（「答錯的題目才是這一章送給您的禮物」）
2. **稱呼讀者用「您」**
3. **比專業名詞更多的說明與舉例**：每個英文名詞第一次出現，立即用情境例子＋日常例子（手搖飲、餐廳、便利商店、打電話）。比喻之後一定回到正式技術意義
4. **去 AI 味（Humanizer 規則）**：
   - 禁用：「此外」「至關重要」「凸顯」「彰顯」「深入探討」「不僅…而且…」「這不只是…而是…」
   - 不用破折號串接句子；少用粗體堆疊；避免固定三項式列舉
   - 句子長短交錯，段落結尾方式要有變化，結尾不喊空泛口號
   - 允許口語化的過場（「就這麼一問一答」「原理就在這裡」）
5. **中立案例原則**：正文用角色名稱（CRM System、Marketing Platform、Identity Provider…）與虛構公司 **Northstar Electronics**；產品名稱只出現在「延伸對照（選讀）」卡
6. 範例程式/流程控制在 5–15 行內，逐步解說，不可貼了不講
