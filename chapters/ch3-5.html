Margin、Padding
與版面

預估 30 分鐘 ・ 建議先讀完 3-4 再來。

上一章教了 CSS 怎麼透過 Selector 套用樣式。這一章要教兩個最常被提到、也最常被搞混的版面觀念:Margin 和 Padding,它們決定了元素之間「留多少空隙」。

  • 分辨 Margin 和 Padding 的差異。
  • 理解 Width、Height、Display 這幾個基本版面屬性的作用。
  • 看到「欄位擠在一起」的問題時,知道大概是哪個屬性沒設好。
情境

「這兩個欄位擠在一起,好像沒有留空間」

Northstar Electronics 的報名表單,顧問發現姓名和 Email 兩個欄位緊貼在一起,回報給工程師:

「這兩個欄位中間完全沒有空隙,擠成一團,可以加一點間距嗎?」工程師回覆:「好,我調一下 margin。」

「調一下 margin」到底在調什麼?這一章要把這個常見詞彙,還有它容易跟另一個詞(padding)搞混的地方講清楚。

情境例子 1/2

相框裡的留白,和相框外的距離

想像一張照片放進相框裡展示。相框內緣和照片之間,通常會留一圈白邊,讓照片不會頂到相框邊緣——這圈白邊,就是 Padding(內距),是元素「內容」和它自己「邊界」之間的空間。

而這個相框掛在牆上時,跟旁邊其他相框之間也會留一些距離,不會緊貼在一起——這段距離,就是 Margin(外距),是這個元素跟「其他元素」之間的空間。

先記住這句 Padding 是「相框內部」的留白(元素內容跟自己邊界的距離),Margin 是「相框跟相框之間」的留白(元素跟其他元素的距離)。
情境例子 2/2

相框的尺寸,和牆面怎麼擺放

除了留白,相框本身還有尺寸——多寬多高,對應到 CSS 的 Width(寬度) 和 Height(高度) 屬性,決定這個元素本身佔多大空間。

還有一個決定「這面牆上的相框怎麼擺放」的屬性,叫 Display(顯示方式)——例如相框是一個一個往下疊(常見設定是 block,像 3-2 教過的 div 預設會這樣排列),還是並排放在同一排(常見設定是 inline,像 span 預設會這樣排列)。

回到開頭姓名和 Email 欄位擠在一起的問題。工程師說要調 margin,意思是要在這兩個欄位「之間」增加空隙,讓它們不再緊貼彼此——這正是 Margin(元素跟其他元素的距離)該負責的事,而不是 Padding(元素內部的留白)。

正式定義

幾個關鍵屬性的正式說法

屬性定義
Margin

串起來看

套用在表單欄位上的實際樣子

接續 3-3 的報名表單,加上版面相關的 CSS:

HTML:
<input type="text" name="firstName" class="form-field">
<input type="text" name="lastName" class="form-field">

CSS:
.form-field {
  margin-bottom: 12px;   /* 每個欄位下方留 12px 的外距,避免緊貼下一個欄位 */
  padding: 8px;          /* 欄位內文字與邊框之間留 8px 的內距,方便閱讀 */
  width: 100%;           /* 欄位寬度佔滿容器 */
}

如果一開始 margin-bottom 沒有設定(或被設成 0),兩個欄位之間就會完全沒有空隙,呈現出開頭情境裡「擠在一起」的樣子。這正是工程師說要調整 margin 來解決的問題。

為什麼要學這個

兩個馬上用得到的場景

場景一:更精準地描述版面問題

與其說「這裡看起來很擠」,不如說「這兩個欄位之間好像沒有留 margin」。即使您不會親自調整,這種說法能讓工程師快速理解問題出在哪個層面,加快溝通效率。

場景二:理解「響應式設計」为什么需要額外考量

同樣的 Margin、Padding、Width 設定,在電腦大螢幕上看起來剛好,但在手機小螢幕上可能顯得過大或版面跑掉。理解這一點,能幫助您在需求討論時,主動提醒「這個版面在手機上會不會需要額外調整」。

核心概念 1/2

Margin 和 Padding,方向不同

概念一:Padding 影響「元素內部」,Margin 影響「元素之間」

如果想讓文字離按鈕邊框遠一點(按鈕看起來更大氣),要調 Padding;如果想讓兩個按鈕之間留一些空隙(不要緊貼),要調 Margin。方向不同,調整的屬性也不同。

概念二:兩者都可以個別針對上下左右四個方向設定

不一定要四個方向都留一樣的空間,常常只需要調整某一個方向(例如只加大下方的 Margin),就能解決版面擁擠的問題,不需要整體重新設計。

核心概念 2/2

Display 決定元素怎麼排列

概念三:block 元素預設獨佔一整行,inline 元素預設並排顯示

3-2 提過 div 是 block(通常換行呈現),span 是 inline(通常不換行、跟文字並排)。這個預設行為背後,就是 Display 屬性在起作用,也可以透過 CSS 手動調整改變。

概念四:版面問題不一定要立刻懷疑資料或程式邏輯

很多「畫面看起來怪怪的」問題,其實只是 Margin、Padding、Width 這幾個基本屬性沒設定好,是相對單純的樣式調整,不需要一開始就懷疑複雜的程式或資料問題。

容易混淆

一組最容易搞混的概念

容易混淆的一組怎麼分
Margin vs PaddingMargin 是元素跟「外面其他元素」的距離(相框跟相框之間);Padding 是元素跟「自己內部內容」的距離(相框內緣跟照片之間)。方向相反,調整的目的也不同。

這一組概念光用文字說明,有些人還是會覺得模糊,沒關係,下次遇到實際案例時,再回頭看這張對照,通常很快就能抓到感覺。

案例

Northstar Electronics 的表單欄位擁擠事件

追查報名表單欄位擠在一起的問題,發現:

CSS 規則:.form-field { padding: 8px; }
缺少的設定:沒有設定 margin-bottom,導致欄位之間完全沒有外部間距
  1. 有哪些角色?表單欄位的 HTML、對應的 CSS 樣式規則。
  2. 資料如何移動?這裡沒有資料流動,單純是版面呈現的樣式設定問題。
  3. 各屬性負責什麼?Padding 已經正確設定,讓欄位內的文字有適當留白;但 Margin 沒有設定,導致欄位與欄位之間緊貼在一起。
  4. 哪裡可能出錯?工程師可能只設定了 Padding,忘記同時考慮 Margin,兩者容易被搞混或遺漏其中一個。
  5. 第一個合理的檢查方向?用開發者工具檢查這個欄位的 CSS 設定,確認 margin-bottom(或其他方向的 margin)是否有正確的數值。

常見誤解

三個很多人踩過的坑

誤解一:「Margin 和 Padding 是同一件事,叫法不同而已」

兩者方向完全不同:Padding 是元素內部的留白,Margin 是元素跟其他元素之間的留白,調整的目的不一樣。

誤解二:「版面擁擠一定要重新設計整個頁面」

很多版面擁擠的問題,只需要調整某個元素的 Margin 或 Padding 數值,不需要大規模重新設計。

誤解三:「電腦上看起來正常,手機上也一定正常」

不同螢幕尺寸下,同樣的版面設定可能呈現完全不同的效果,這也是為什麼響應式設計需要額外針對不同裝置做調整與測試。

遇到問題先查什麼

「版面看起來擁擠或跑版」的檢查順序

  1. 用開發者工具檢查出問題的元素。看它目前的 Margin 和 Padding 數值。
  2. 確認是「元素內部」擁擠,還是「元素之間」擁擠。前者該查 Padding,後者該查 Margin。
  3. 確認 Width 或 Height 是否設定合理。元素本身的尺寸是否過大或過小。
  4. 換到不同螢幕尺寸測試。確認是否為響應式設計未涵蓋到的情況。
  5. 把具體的畫面現象和螢幕尺寸提供給工程師。
重點整理

帶走這四句話

  1. Margin 是元素跟其他元素之間的外部空間,Padding 是元素跟自身內容之間的內部空間。
  2. Width、Height 決定元素本身的尺寸,Display 決定元素如何排列呈現。
  3. 版面擁擠或跑版,經常只是這幾個基本屬性沒設定好,不一定是複雜的問題。
  4. 響應式設計讓版面能因應不同螢幕尺寸調整,同樣的設定在不同裝置上可能呈現不同效果。
檢核 1/4

想讓兩個相鄰的按鈕之間留一點空隙,不要緊貼在一起,應該調整哪個屬性?

正確答案是 B。Margin 是元素跟其他元素之間的外部空間,調整它才能讓相鄰元素之間留出空隙。
檢核 2/4

是非題:Padding 影響的是元素內容跟自身邊界之間的空間,不是跟其他元素的距離。

正確答案是「是」。Padding 是元素邊界與自身內容之間的內部空間,跟其他元素的距離是 Margin 負責的範疇。
檢核 3/4

決定一個元素是否獨佔一整行,還是跟其他內容並排顯示的屬性是什麼?

正確答案是 C。Display 屬性決定元素如何排列呈現,例如 block 獨佔一整行,inline 與其他內容並排。
檢核 4/4

情境題:表單裡兩個欄位緊貼在一起,毫無空隙。第一個合理的檢查方向是?

正確答案是 A。欄位緊貼在一起,最常見的原因是缺少或設錯了 margin,這是相對單純、成本很低的檢查方向。
