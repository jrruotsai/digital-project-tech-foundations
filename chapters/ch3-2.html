常見 HTML 元素
div、a、img 與 iframe

預估 30 分鐘 ・ 建議先讀完 3-1 再來。

上一章教了 Tag、Element、Attribute 這三個框架性的名詞。這一章要認識幾個最常出現的具體元素,讓您看到網頁原始碼時,能認出幾個熟悉的面孔。

  • 認得 div、span、p、a、img 這幾個常見元素的用途。
  • 理解 iframe 和 script 這兩個比較特別的元素在做什麼。
  • 看到一段陌生的 HTML,能大致猜出畫面上有哪些區塊。
情境

「這個嵌入的地圖是怎麼放進我們網頁的?」

Northstar Electronics 的官網「聯絡我們」頁面,嵌入了一張地圖,顧問好奇地問工程師:

「這張地圖看起來是另一個網站的東西,怎麼會出現在我們自己的頁面裡?」

答案跟這一章要教的 iframe 有關——一種可以把「別的網頁」直接嵌進「自己網頁」裡的特殊元素。這一章會把它和幾個更常見的元素一起介紹清楚。

情境例子 1/2

收納箱、文字段落與連結按鈕

想像整理房間時,您會用不同的容器收納不同的東西。div(division,區塊)就像一個萬用收納箱,本身沒有特定含義,單純用來把幾樣東西「框」在一起,方便整理和管理,3-1 案例裡就出現過它。

span則像是在一段文字裡,用螢光筆特別標出的一小段,通常用來對一段文字裡的「某幾個字」做特別的處理,而不是整段。

p(paragraph,段落)專門用來放一段文字內容,例如一段說明或介紹。a(anchor,錨點)則是可以點擊的連結,通常搭配 href 這個 Attribute(3-1 教過),告訴瀏覽器點下去要跳去哪裡。

先記住這句 div 和 span 都是「沒有特定意義的容器」,差別在 div 通常框住一整塊(區塊呈現),span 通常框住一小段文字(不換行呈現)。
情境例子 2/2

相框裡的照片,和窗戶裡的另一個房間

img(image,圖片)就像掛在牆上的相框,顯示一張圖片,搭配 3-1 教過的 src 屬性告訴瀏覽器圖片檔案在哪裡。

iframe(inline frame,行內框架)則特別一些,它像是牆上開了一扇窗戶,窗戶裡看到的,其實是完全獨立的另一個房間(另一個網頁),只是被「嵌入」顯示在您目前這個頁面裡。回到開頭的地圖案例——那張地圖其實是地圖服務商自己的網頁,Northstar 的官網用 iframe 開了一扇窗,把它顯示在自己的頁面裡。

還有一個常見但看不見形體的元素:script,它不顯示任何畫面內容,而是用來放置 JavaScript 程式碼(3-6 會深入教),讓網頁具備互動能力。

正式定義

幾個常見元素的正式說法

元素用途
div沒有特定語意的區塊容器,用來組織版面結構
span沒有特定語意的行內容器,用來標記一小段文字
p一段文字內容
a可點擊的連結,通常搭配 href 屬性指定跳轉目標
img顯示一張圖片,搭配 src 屬性指定圖片來源
iframe在目前頁面中嵌入另一個獨立網頁的內容
script放置 JavaScript 程式碼,不直接顯示內容,但能讓頁面產生互動行為
串起來看

一個聯絡頁面的完整結構

把這些元素組合起來,看一個簡化的「聯絡我們」頁面:

<div>
  <p>歡迎聯絡 Northstar Electronics</p>
  <p>客服信箱:<span>support@northstar-electronics.com</span></p>
  <a href="/contact-form">填寫線上表單</a>
  <iframe src="https://maps.example.com/embed"></iframe>
</div>
<script>
  // 這裡放置追蹤或互動用的程式碼(3-6、3-8 會深入教)
</script>

整個頁面被一個 div 包起來;裡面有兩段文字(p),其中一段用 span 特別標出信箱;一個可點擊的連結(a);一扇嵌入地圖的窗戶(iframe);頁面最後還有一段看不見形體、負責處理程式邏輯的 script。

為什麼要學這個

兩個馬上用得到的場景

場景一:理解「嵌入」和「自己開發」的差異

當客戶要求「把某個外部服務的內容放進我們網頁」,理解 iframe 的原理後,您會知道這通常是「嵌入別人的網頁」,而不是把對方的功能整個搬進自己系統裡重新開發,兩者的開發工作量和維護責任完全不同。

場景二:排查畫面問題時,先確認是哪一種元素

如果嵌入的地圖顯示不出來,問題可能出在 iframe 本身的來源網址,或是對方網站不允許被嵌入,而不是自己網站的問題。分清楚是哪一種元素出狀況,能幫助快速判斷該找誰處理。

核心概念 1/2

有些元素有意義,有些只是容器

概念一:div 和 span 本身沒有語意,意義來自您怎麼使用它

相對於 p(明確代表段落)、img(明確代表圖片),div 和 span 本身不代表任何特定內容類型,純粹是組織版面用的容器,實際用途取決於工程師怎麼安排裡面的內容。

概念二:iframe 裡的內容,是完全獨立的另一個網頁

嵌入的地圖、影片這類內容,運作邏輯跟一個完全獨立的網站一樣,只是被框在您的頁面裡顯示。這也是為什麼 iframe 裡的內容,通常不受您自己網站的樣式或程式直接控制。

核心概念 2/2

script 元素,是通往互動與追蹤的入口

概念三:script 不顯示畫面,但影響力可能很大

script 元素本身在畫面上完全看不到,但它承載的程式碼,可能負責表單驗證(3-7 會教)、追蹤使用者行為(3-8、單元 10 會教),甚至呼叫其他系統的 API(單元 6 會教)。看不見形體,不代表沒有作用。

概念四:認得這幾個元素,能大致「讀懂」陌生網頁的結構

即使不會寫程式,認出這幾個常見元素,再搭配 2-8 教過的開發者工具,您已經具備初步「看懂」一個陌生網頁結構的能力,這對需求訪談和問題排查都很有幫助。

容易混淆

兩組容易搞混的概念

容易混淆的一組怎麼分
div vs span兩者都是無特定語意的容器,差別在 div 通常用來框住一整塊區域(呈現時會自動換行),span 通常用來框住一小段文字(呈現時不換行,融合在原本的文字流裡)。
img vs iframeimg 顯示的是一張靜態圖片檔案;iframe 嵌入的是一整個獨立的網頁,可能包含自己的互動功能(例如可以拖曳縮放的地圖)。
案例

Northstar Electronics 的嵌入地圖失效事件

Northstar 官網的「聯絡我們」頁面,某天嵌入的地圖突然變成一片空白,追查發現:

Northstar 端的 HTML:iframe 的 src 網址沒有任何變動
地圖服務商:當天調整了安全政策,不再允許被其他網站的 iframe 嵌入顯示
  1. 有哪些角色?Northstar 官網(嵌入方)、地圖服務商(被嵌入方)。
  2. 資料如何移動?Northstar 頁面透過 iframe,請求顯示地圖服務商的網頁內容。
  3. 各方負責什麼?地圖服務商決定自己的內容是否允許被其他網站嵌入顯示。
  4. 哪裡可能出錯?如果被嵌入方調整了安全設定,拒絕被嵌入,Northstar 這邊即使程式碼完全沒變,畫面也會出問題——問題出在對方,不是自己網站壞了。
  5. 第一個合理的檢查方向?先確認 Northstar 自己最近有沒有修改過相關程式碼;如果沒有,就該懷疑是被嵌入的第三方服務,自己調整了設定或政策。

通用概念常見情境

常見誤解

三個很多人踩過的坑

誤解一:「div 和 span 有特定的意思,一定要照規定用」

兩者本身沒有特定語意,是通用容器,實際安排完全取決於工程師怎麼組織版面,沒有「正確用法」這種嚴格規定。

誤解二:「iframe 裡的內容,是我們自己網站的一部分」

iframe 嵌入的通常是完全獨立的另一個網頁,對方的政策或設定改變,可能導致嵌入內容失效,這不代表自己網站的程式碼出了問題。

誤解三:「看不到 script 元素的畫面效果,就代表它沒有作用」

script 本身不顯示畫面內容,但可能在背景執行追蹤、驗證等重要邏輯,不能因為看不見就忽略它的存在與影響。

遇到問題先查什麼

「嵌入內容顯示不出來」的檢查順序

  1. 確認這是哪一種元素造成的。用開發者工具檢查,是 img 還是 iframe?
  2. 確認自己這邊最近有沒有修改過相關程式碼。排除自己端造成問題的可能。
  3. 確認來源網址(src)是否正確、是否還有效。
  4. 如果是 iframe,確認對方網站是否調整了安全政策。這類問題通常需要聯繫對方確認。
  5. 把觀察到的具體現象(哪個元素、什麼時候開始)提供給工程師。
重點整理

帶走這四句話

  1. div 和 span 是無特定語意的通用容器,前者框住區塊,後者框住行內文字。
  2. p、a、img 各自有明確用途:段落文字、可點擊連結、顯示圖片。
  3. iframe 是把另一個獨立網頁嵌入顯示的窗口,script 是不顯示畫面但承載程式邏輯的元素。
  4. 認得這幾個常見元素,能幫助您初步讀懂陌生網頁的結構,對訪談和排查都很有幫助。
檢核 1/4

下列哪一個元素,是用來嵌入另一個完全獨立網頁的內容?

正確答案是 B。iframe 就像牆上的一扇窗,讓您能把另一個獨立的網頁,嵌入顯示在目前的頁面裡。
檢核 2/4

是非題:div 和 span 本身沒有特定的語意,是通用的容器元素。

正確答案是「是」。div 和 span 都是無特定語意的通用容器,實際用途完全取決於工程師如何安排內容。
檢核 3/4

負責放置 JavaScript 程式碼、本身不顯示任何畫面內容的元素是哪一個?

正確答案是 C。script 元素不顯示畫面內容,但承載 JavaScript 程式碼,讓網頁具備互動能力。
檢核 4/4

情境題:官網嵌入的第三方地圖突然顯示空白,但自己的程式碼完全沒有變動。第一個合理的懷疑方向是?

正確答案是 B。iframe 嵌入的是對方獨立的網頁,如果自己的程式碼沒有變動,問題很可能出在被嵌入方調整了設定或政策。
