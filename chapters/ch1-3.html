Website、Web App 與 Mobile App
有什麼不同

預估 30 分鐘 ・ 建議先讀完 1-1、1-2 再來。

「我們官網要不要順便做個 App？」這句話,您在專案裡大概聽過不只一次。今天要把 Website、Web App、Mobile App 這三個詞分清楚,讓您下次聽到類似的問題,心裡先有個底。

  • 用自己的話說出三者的差異。
  • 判斷一個需求適合做成哪一種形式。
  • 知道「手機也能用」背後可能藏著哪些不同的工作量。
情境

「我們要不要也做一個 App？」

客戶的行銷主管在會議上說:

「競爭對手都有自己的 App 了,我們是不是也該做一個?顧客可以在上面查訂單、看優惠。」

這句話聽起來合理,但作為顧問,您需要先搞清楚一件事:客戶想要的,究竟是一個「手機上也能開的網頁」,還是一個「要下載安裝的獨立程式」?這兩者從開發方式、上架流程到後續維護,幾乎是兩個世界。

先別急著回答「好啊」,我們先把這三個詞的差異搞清楚。

情境例子 1/2

逛街,還是去自己家的便利商店

Website(網站)就像一條商店街。您不用辦會員、不用下載任何東西,走過去、推門進去就能逛。您在 Browser 裡輸入網址,就是「走進」了這家店。逛完關掉分頁,下次再逛,還是同一個方式進去。

公司的形象官網、活動報名頁面,大部分都是 Website。它的核心工作是「呈現資訊」,讓訪客瀏覽、閱讀、填表單。

Web App(網頁應用程式)像是同一條商店街裡,一間可以讓您「辦事」的店,不只是逛,還能存提款、辦手續。技術上它一樣是在 Browser 裡打開的網頁,但功能更複雜:登入、查詢、送出申請、即時互動。

先記住這句 Website 和 Web App 技術上都活在 Browser 裡,差別在「功能複雜度」,不是「用不用瀏覽器」。一個是逛街看看,一個是進去辦正事。
情境例子 2/2

而 Mobile App,是您自己家開的店

Mobile App(行動應用程式)不一樣。它不是走進別人的商店街,而是您自己在手機裡「開了一家專屬店面」,平常收在口袋裡(桌面圖示),想用的時候直接點開,不用先經過瀏覽器這條街。

因為是自己的店面,它可以做到 Browser 裡的網頁做不到的事:離線也能用某些功能、推播通知直接跳到手機鎖定畫面、呼叫手機的相機和 GPS。代價是:顧客要先「搬家過來」,也就是到 App Store 或 Google Play 下載安裝。

回到會議室的問題。「顧客可以查訂單、看優惠」,這種需求多半 Web App 就能滿足,不一定要蓋一整間新店面。但如果客戶要的是「即使沒網路也能用」或「常態性推播提醒」,那可能真的要考慮 Mobile App 了。

正式定義

三個名詞的正式說法

名詞定義典型例子
Website
網站		在 Browser 中開啟、以呈現資訊為主的網頁集合,不需要安裝公司官網、活動報名頁
Web App
網頁應用程式		在 Browser 中開啟、但具備登入、互動、資料處理等應用程式功能的網頁線上客服系統、CRM 的網頁後台
Mobile App
行動應用程式		需要從應用程式商店下載安裝、直接執行在手機作業系統上的程式銀行 App、社群媒體 App

三者的共通點:核心都是「前端呈現 + 後端邏輯 + 資料庫」,這是 1-1 教過的三層架構。不同的是 Client 的形式,這是 1-2 教過的角色分類。您已經有的兩把刀,這裡剛好都能派上用場。

串起來看

同一份資料,三種入口

Northstar Electronics 的客戶想查自己的訂單狀態,可以有三種入口:

入口一(Website 情境):客戶到活動頁登記,只是留資料,不需要登入查詢
入口二(Web App 情境):客戶在 Browser 開啟「會員中心」網頁,登入後查訂單
入口三(Mobile App 情境):客戶打開手機上的 Northstar App,查訂單、收到出貨推播

後端和資料庫可能是同一套,三種入口只是不同的 Client 在向同一個 Server 要資料。這也呼應 1-2 教的:Client 的形式可以不同,一問一答的本質不變。

為什麼要學這個

估工時、對需求,都靠這個分清楚

場景一:需求訪談時聽懂真正的規模

客戶說「手機也能用」時,先反問:「是希望手機瀏覽器打開好用就好,還是需要下載安裝的獨立 App?」前者可能只是網頁加上一些排版調整(響應式設計,後面單元 3 會教),後者是另立一個開發、上架、維護的專案。

場景二:判斷技術限制

如果客戶要「離線時也能記錄資料,之後回到有網路的地方自動同步」,這種功能通常只有 Mobile App 做得到,單純的 Web App 很難做到完整的離線體驗。懂得這個界線,才能及早提醒客戶,不會等到開發到一半才發現做不到。

核心概念 1/2

形式不同,骨架相同

概念一:三者都是 Client,只是形式不同

不管是 Website、Web App 還是 Mobile App,面對後端的 Server 時,它們都是發出請求的 Client(1-2 教過的角色)。差別只在「用什麼工具發問」:瀏覽器,還是獨立安裝的程式。

概念二:安不安裝,是最大的分界線

Website 和 Web App 都不用安裝,打開網址就能用,這是它們最大的共同點。Mobile App 需要安裝,這件事看似小,卻決定了完全不同的散布方式(要說服使用者下載)、更新方式(App 商店審核)和維護成本。

核心概念 2/2

功能複雜度,和裝置能力

概念三:Website 到 Web App 是「功能」的光譜,不是「是非題」

很多網站介於兩者之間:大部分頁面是靜態呈現(Website 性質),但有個會員登入區塊做得很像應用程式(Web App 性質)。與其糾結該叫哪個名字,不如直接描述「這個頁面需要登入和互動」比較實際。

概念四:只有 Mobile App 能完整用到裝置的硬體能力

推播通知、離線儲存、呼叫相機和 GPS,這些是手機作業系統開放給已安裝 App 的能力。Browser 裡的網頁能做到一部分(例如網頁也能推播通知),但完整、穩定的體驗,通常還是 Mobile App 比較可靠。

容易混淆

兩組容易搞混的概念

容易混淆的一組怎麼分
Web App vs Mobile App都能登入、互動、處理資料。差別在要不要安裝:Web App 打開瀏覽器就能用,Mobile App 要先下載安裝到手機。
「響應式網站」vs「Mobile App」響應式網站是同一個 Website,在手機瀏覽器上自動調整版面,本質還是 Website;Mobile App 是完全獨立安裝的程式。客戶說「手機也能用」時,先確認是指哪一種。
案例

Northstar Electronics 的三個入口決策

Northstar Electronics 的行銷團隊想推出一個「會員專屬優惠」功能,列出三個選項讓您評估:

選項 A:在官網加一個會員登入頁,登入後看到專屬優惠(Web App 性質)
選項 B:做一個獨立的 Mobile App,推播優惠通知
選項 C:兩者都做,分階段上線
  1. 有哪些角色?行銷團隊(需求方)、開發團隊、最終的會員客戶。
  2. 資料如何移動?不管哪個入口,最終都要向同一套後端和資料庫要優惠資料。
  3. 各選項負責什麼?選項 A 開發快、不用上架審核;選項 B 能做推播,但多一層 App 商店的審核與維護。
  4. 哪裡可能出錯?如果客戶一開始只說「要有優惠功能」,沒說清楚是否要推播,雙方可能對工時和範圍有完全不同的預期。
  5. 第一個合理的檢查方向?反問客戶:「這個功能一定要能推播通知嗎?」這一句就能決定要不要往 Mobile App 的方向討論。

常見誤解

三個很多人踩過的坑

誤解一:「有網站就等於有 App 的功能」

網站可以做出很多互動功能,但推播通知、完整的離線使用,通常還是 Mobile App 的專長。不是網站做得不夠好,是兩種形式本來就有不同的能力邊界。

誤解二:「做 App 一定比做網站更專業、更值得投資」

App 需要使用者先下載安裝,這其實是一道不小的門檻。如果客戶的使用情境是「久久用一次」,勉強推 App 反而可能沒人下載,網頁形式的觸達率通常更高。

誤解三:「響應式網站就是 Mobile App」

響應式網站只是同一個網站在手機上自動調整版面,骨子裡還是 Website,不會出現在使用者的桌面圖示上,也不能推播通知。

遇到問題先查什麼

「客戶要手機也能用」的釐清順序

  1. 確認使用情境。使用者會頻繁使用,還是偶爾用一次?
  2. 確認是否需要離線功能。沒有網路時,還需要能操作嗎?
  3. 確認是否需要推播通知。這是 Mobile App 的強項,Website/Web App 較難做到穩定體驗。
  4. 確認是否需要呼叫裝置硬體。相機、GPS、生物辨識這類功能,通常需要 Mobile App。
  5. 把答案回饋給客戶,說明對應的形式與大致工作量差異。讓客戶自己確認要的是哪一種。
重點整理

帶走這四句話

  1. Website、Web App、Mobile App 都是 Client,差別在形式:要不要瀏覽器、要不要安裝。
  2. Website 偏呈現資訊,Web App 偏處理事情,但兩者都活在 Browser 裡,不用安裝。
  3. Mobile App 需要安裝,換來的是推播、離線、裝置硬體這些額外能力,也換來上架與維護的額外成本。
  4. 客戶說「手機也能用」時,先釐清情境,再決定形式,別急著承諾要做 App。
檢核 1/4

下列哪一項是 Mobile App 和 Web App 最大的分界?

正確答案是 B。Web App 打開瀏覽器就能用,Mobile App 需要先下載安裝到裝置上,這是兩者最根本的差異。
檢核 2/4

是非題:公司官網加上「響應式設計」讓手機瀏覽器也好看,這樣就算完成了一個 Mobile App。

正確答案是「否」。響應式設計只是同一個 Website 在手機瀏覽器上調整版面,骨子裡還是 Website,不會出現在桌面圖示,也不能推播通知。
檢核 3/4

客戶要求「使用者沒有網路時,還要能繼續填寫巡檢表單,回到有網路的地方自動同步」。這個需求最適合哪種形式?

正確答案是 C。完整穩定的離線使用能力,是 Mobile App 的強項,一般網頁形式很難做到同等的離線體驗。
檢核 4/4

情境題:客戶說「希望手機也能用」。您的第一步應該是?

正確答案是 B。先釐清客戶真正需要的能力,才能判斷該用哪種形式,避免雙方對工作量有完全不同的想像。
