CSS 如何控制樣式
Selector、Class 與 ID

預估 30 分鐘 ・ 建議先讀完 3-1、3-3 再來。

3-1 提過「HTML 只負責結構,不負責好不好看」。這一章要介紹負責外觀的另一半:CSS。它決定了文字的顏色、按鈕的大小、頁面的排版。

  • 說明 CSS 在網頁裡的角色。
  • 看懂一條簡單的 CSS 規則在做什麼。
  • 分辨用 Class 和用 ID 設定樣式的差異。
情境

「同樣的按鈕,怎麼有的紅有的藍?」

Northstar Electronics 的官網改版後,顧問發現同一種「送出」按鈕,在不同頁面顏色不一致,詢問工程師:

「這幾個按鈕的 HTML 結構應該一樣吧?為什麼顏色不一致?」工程師回答:「可能有些頁面套用了舊的樣式規則,沒有更新到最新的 class。」

這句話牽涉到這一章要教的核心概念:同樣的 HTML 元素,顯示成什麼樣子,取決於套用了哪一條 CSS 規則。這一章要把這個運作原理講清楚。

情境例子 1/2

裝潢公司的統一規格表

想像一家連鎖店要裝潢好幾家分店,總部訂了一份規格表:「所有招牌用紅色底、白色字」「所有收銀台用同一種木紋」。裝潢師傅照著規格表施工,不用每家店重新設計一次。

CSS(Cascading Style Sheets,階層式樣式表)就是這份規格表,它獨立於 HTML 結構之外,專門定義「哪些元素該長什麼樣子」。這樣的分工讓同一份 HTML 結構,可以透過套用不同的 CSS,呈現完全不同的外觀。

先記住這句 HTML 負責「這裡有什麼」,CSS 負責「這個東西長什麼樣子」。同一段 HTML,套用不同的 CSS,外觀可以完全不同。
情境例子 2/2

規格表怎麼知道要套用在哪裡

規格表要能運作,必須先講清楚「這條規則,適用於哪些東西」。例如「所有招牌」「這一家特定分店的收銀台」。這個「指定套用對象」的機制,叫做 Selector(選擇器)。

最常見的兩種選擇方式:一種是靠 3-3 教過的 class,像是規格表寫「所有貼了『分店招牌』標籤的東西,都套用這條規則」——同一個 class 可以套用在很多個元素上,一次改規則,全部一起變。另一種是靠 id,像是規格表寫「只有『忠孝店的收銀台』這一個特定位置,套用這條特別規則」——id 是唯一的,只精準指定某一個元素。

回到開頭的按鈕顏色問題。如果按鈕原本統一透過某個 class 套用顏色規則,但某些頁面的按鈕忘記加上這個 class(或用了一個舊的 class 名稱),就會呈現不一致的顏色——問題不在 HTML 結構本身,而在於「有沒有正確連結到該套用的樣式規則」。

正式定義

幾個關鍵名詞的正式說法

名詞定義
CSS獨立於 HTML 結構之外,用來定義畫面外觀樣式的語言
Selector
選擇器		指定一條 CSS 規則要套用在哪些 HTML 元素上的寫法
Class Selector透過 class 屬性,把規則套用到所有具有相同 class 的元素上
ID Selector透過 id 屬性,把規則精準套用到某一個特定元素上

看一個實際範例:

.submit-button {
  background-color: blue;
  color: white;
}

開頭的 .submit-button 就是一個 Class Selector(前面的點是慣例寫法,代表這是套用在 class 上),意思是「所有 class 為 submit-button 的元素,背景色設為藍色、文字設為白色」。

串起來看

HTML 和 CSS 怎麼連結起來

接續 3-3 教過的報名表單,加上 CSS 樣式:

HTML:
<button class="submit-button" type="submit">Submit</button>

CSS:
.submit-button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
}

HTML 裡的按鈕帶著 class="submit-button";CSS 裡有一條規則,專門指定「所有帶著這個 class 的元素」該長什麼樣子。瀏覽器讀取網頁時,會把兩邊對照起來,套用對應的樣式。如果某個頁面的按鈕忘了加上這個 class,瀏覽器就找不到該套用哪條規則,只會顯示預設的樣子(通常是很陽春的黑白按鈕)。

為什麼要學這個

兩個馬上用得到的場景

場景一:排查「畫面樣式不一致」的問題

發現某個頁面的元素樣式跟其他頁面不同,可以請工程師確認這個元素的 class 是否正確,或者對應的 CSS 規則是否有被正確載入。這通常比直接說「這裡看起來怪怪的」更能幫助工程師快速定位。

場景二:理解「改樣式」和「改內容」是兩件不同的事

客戶要求「把按鈕改成綠色」,這通常只需要調整 CSS,不需要更動 HTML 結構,是相對輕量的改動;但如果要求「多一個欄位」,就牽涉到 HTML 結構的變化,兩者的工作量和風險並不相同。

核心概念 1/2

CSS 讓「內容」和「外觀」分開管理

概念一:同一份 HTML,可以套用不同的 CSS,呈現不同外觀

這種分離的設計,讓網站可以做「改版」而不用重寫整個內容結構,也讓多個頁面能透過共用同一份 CSS,維持一致的視覺風格。

概念二:Class Selector 用來做「一批」元素的統一設定

如果同一種樣式要套用到很多個元素上(例如所有的按鈕、所有的標題),用 class 是最有效率的做法——改一條規則,全部套用的元素都會一起改變。

核心概念 2/2

ID Selector 用來做「唯一」的精準設定

概念三:ID Selector 適合用在只有單一個特例的情況

如果某個元素需要一個跟其他人都不一樣、獨一無二的特殊樣式,用 id 來指定會更精準。但因為 id 理論上不能重複,不適合用在需要套用到多個元素的情境。

概念四:找不到對應的 Selector,元素就只會顯示瀏覽器的預設樣式

如果 HTML 元素的 class 或 id 打錯字,或是對應的 CSS 規則沒有被正確載入,瀏覽器不會報錯,只會安靜地顯示最陽春的預設樣子。這也是為什麼樣式問題有時候很難第一眼就看出原因。

容易混淆

兩組容易搞混的概念

容易混淆的一組怎麼分
Class Selector vs ID SelectorClass 可以套用在多個元素上,適合統一管理一批相似的元素;ID 只能對應到唯一一個元素,適合處理特殊、唯一的情況。
Inline CSS vs External CSSInline CSS 是直接寫在 HTML 元素的 style 屬性裡,只影響這一個元素;External CSS 是寫在獨立的樣式表檔案裡,可以被多個頁面共用。維護大型網站時,通常偏好使用 External CSS,方便統一管理與更新。
案例

Northstar Electronics 的按鈕顏色不一致事件

追查官網送出按鈕顏色不一致的問題,發現:

正常頁面的按鈕:<button class="submit-button">
異常頁面的按鈕:<button class="submit-btn">(class 名稱少打了一個字,與 CSS 規則對不上)
  1. 有哪些角色?各個頁面的 HTML(使用按鈕的地方)、統一管理的 CSS 樣式表。
  2. 資料如何移動?這裡沒有系統間的資料流動,是同一個網站內,HTML 的 class 屬性要正確對應到 CSS 規則的名稱。
  3. 各部分負責什麼?CSS 規則定義了 submit-button 這個 class 該長什麼樣子,只有正確使用這個名稱的元素,才能套用到規則。
  4. 哪裡可能出錯?異常頁面的按鈕,class 名稱打錯字(少了一個字母),導致瀏覽器找不到對應的樣式規則,只顯示預設外觀。
  5. 第一個合理的檢查方向?用開發者工具檢查異常頁面按鈕的 class 名稱,跟正常頁面逐字比對,確認是否有拼字差異。

常見誤解

三個很多人踩過的坑

誤解一:「同一個元素樣式跑掉,一定是程式邏輯壞了」

很多時候只是 class 名稱打錯字,或對應的樣式表沒有被正確載入,不一定是複雜的程式邏輯問題。

誤解二:「改樣式跟改內容一樣複雜」

調整 CSS 通常比調整 HTML 結構或後端邏輯更輕量,兩者的工作量和風險不能一概而論。

誤解三:「id 和 class 可以隨便換著用,沒差別」

id 理論上是唯一的,如果被誤用在需要套用到多個元素的情境,可能導致樣式規則衝突或無法正確套用。

遇到問題先查什麼

「畫面樣式不一致」的檢查順序

  1. 用開發者工具檢查出問題的元素。看它的 class 或 id 是什麼。
  2. 跟正常顯示的頁面逐字比對名稱。確認是否有拼字或大小寫差異。
  3. 確認對應的 CSS 樣式表是否有被正確載入。可能整份樣式表都沒載入成功。
  4. 確認是不是同時套用了多條互相衝突的規則。不同規則可能互相覆蓋。
  5. 把具體差異(class 名稱、樣式表載入狀況)提供給工程師。
重點整理

帶走這四句話

  1. CSS 是獨立於 HTML 之外,專門定義外觀樣式的語言。
  2. Selector 決定一條 CSS 規則要套用在哪些元素上,常見的有 Class Selector 和 ID Selector。
  3. Class 可以套用在多個元素上,適合統一管理;ID 是唯一的,適合處理特殊情況。
  4. 畫面樣式跑掉,經常是 class 名稱對不上或樣式表沒載入,不一定是複雜的程式問題。
檢核 1/4

負責定義網頁外觀樣式(顏色、大小)的語言是什麼?

正確答案是 C。CSS 是專門用來定義網頁外觀樣式的語言,HTML 負責的是內容結構。
檢核 2/4

是非題:同一個 class,可以同時套用在多個不同的 HTML 元素上。

正確答案是「是」。class 可以套用在多個元素上,方便統一管理一批相似元素的樣式。
檢核 3/4

想要精準指定「只有這一個」特定元素的獨特樣式,比較適合用哪一種 Selector?

正確答案是 B。ID Selector 適合用在只有單一個特例、獨一無二的情況,因為 id 理論上不會重複。
檢核 4/4

情境題:某個頁面的按鈕顏色跟其他頁面不一致,結構看起來相同。第一個合理的檢查方向是?

正確答案是 A。樣式不一致經常是 class 名稱對不上造成的,用開發者工具比對是最直接、成本最低的檢查方式。
