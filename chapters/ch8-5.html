Device Identifier 與 User Identifier — 認裝置還是認人

預估 30 分鐘 ・ SDK、套件與應用程式

Device Identifier 辨識裝置，User Identifier 辨識登入者；登出、換機與多人共用裝置時要重新關聯。

  • 能說明 Device Identifier：辨識裝置或安裝實體的識別值。
  • 能區分 Device Identifier 與 User Identifier 在流程中的用途。
  • 遇到問題時，先核對 SDK 版本與相容性。
案例起點

Device Identifier 和 User Identifier 沒有寫清楚時

Northstar 的規格同時提到 Device Identifier 與 User Identifier，卻沒有標出兩者的適用條件與處理邊界。

Device Identifier：辨識裝置或安裝實體的識別值。；User Identifier：辨識登入者或業務使用者的識別值。

測試時要避免這個結果：把裝置當成人，會把前一位使用者的行為算到下一位身上。

概念拆解 1/2

Device Identifier 的處理邊界

Device Identifier：辨識裝置或安裝實體的識別值。

套用到本章情境，第一步是核對 SDK 版本與相容性。確認 Device Identifier 的適用範圍後，再檢查 User Identifier，不從畫面直接猜原因。

判斷重點說明 Device Identifier 時，要交代誰提供輸入、系統套用哪條規則，以及會產生什麼結果。
概念拆解 2/2

User Identifier 與 Device Identifier 的差別

User Identifier：辨識登入者或業務使用者的識別值。

Device Identifier 辨識裝置，User Identifier 辨識登入者；登出、換機與多人共用裝置時要重新關聯。

不要混用Device Identifier 與 User Identifier 不能互換；規格要分別寫出兩者的輸入、處理方式與輸出。
精確定義

本章名詞的工作定義

術語定義判斷時要問
Device Identifier辨識裝置或安裝實體的識別值。它接收什麼輸入，完成後留下什麼結果？
User Identifier辨識登入者或業務使用者的識別值。它和 Device Identifier 的邊界在哪裡？
Evidence能證明 Device Identifier 或 User Identifier 實際結果的原始資料，例如 Request／Response、Log、時間、版本與識別值。哪一筆原始紀錄能支持目前的判斷？

定義 Device Identifier 與 User Identifier 時，必須寫出用途、處理邊界和可觀察結果。

流程位置

Device Identifier 在整體流程中的位置

App → SDK Initialization → Configuration → Event → Callback／Log
→ 觀察結果與例外
→ 記錄負責人與處理決定

沿著流程逐段比對輸入與輸出。若 Device Identifier 的輸入正確而 User Identifier 的輸出不符預期，就從兩者之間的邊界開始查。

專案應用

這個概念會出現在哪些工作？

規格與設計

App 顧問需要把「Device Identifier 與 User Identifier — 認裝置還是認人」改寫成可確認的規則，放進設定清單與裝置 Log。

問題排查

先核對 SDK 版本與相容性，再確認 Configuration 與 Environment。這能分辨問題是否位於 Device Identifier 或 User Identifier。

核心概念 1/2

輸入、規則與輸出要分開寫

輸入

列出 Device Identifier 開始前已知的資料、身分、事件或設定。輸入不明，後面的結果就無法比較。

規則與輸出

User Identifier 執行後，保存實際套用的規則版本、Response、狀態或資料值。不要用「看起來正常」代替可查證結果。

核心概念 2/2

成功條件和失敗條件都要具體

成功條件

Device Identifier 辨識裝置，User Identifier 辨識登入者；登出、換機與多人共用裝置時要重新關聯。 驗收 Device Identifier 與 User Identifier 時，寫明預期狀態、資料值與完成時間。

失敗條件

把裝置當成人，會把前一位使用者的行為算到下一位身上。 Device Identifier 或 User Identifier 失敗時，保留原始輸入、環境、時間與識別值。

容易混淆

用處理邊界分清楚三組概念

容易混淆實際差別
Device Identifier vs User IdentifierDevice Identifier：辨識裝置或安裝實體的識別值。 User Identifier：辨識登入者或業務使用者的識別值。
規格存在 vs 實際生效Device Identifier 的文件是預期規則；User Identifier 的設定值、Response 或 Log 才能證明實際結果。
單次成功 vs 穩定可重現Device Identifier 單次成功不代表 User Identifier 在其他條件下也正確；要用相同輸入重做。

內容文件中的寫法

常見誤解

三個會讓排查走錯方向的假設

把 Device Identifier 和 User Identifier 當成同一件事

只看最後畫面

畫面是流程的最後一層。仍要核對 Device Identifier 的輸入與 User Identifier 的執行紀錄。

一次改多個條件

同時修改 Device Identifier 與 User Identifier 的條件，即使恢復也無法歸因。每次只改一項。

排查順序

Device Identifier 發生異常時的排查順序

  1. 1. 核對 SDK 版本與相容性。
  2. 2. 確認 Configuration 與 Environment。
  3. 3. 查 Initialization、權限與網路。
  4. 4. 確認 Device／User Identifier 與事件內容。
  5. 5. 查看 Callback、Application Log 與後台資料。

每一步先記錄原始值再做下一步。若修改 Device Identifier 或 User Identifier 的設定，保存修改前後值與時間。
