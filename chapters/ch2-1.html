網站網址與 DNS
網址是怎麼變成伺服器的

預估 30 分鐘 ・ 建議先讀完單元 1,特別是 1-2 的 Client/Server 觀念。

單元 1 我們學了「Client 發問、Server 回答」。這一章要補上中間缺的一塊:您在網址列打的那串字,到底是怎麼找到正確的 Server 的?讀完這一章,您應該可以:

  • 拆解一個網址的組成部分。
  • 用自己的話說明 IP、Domain、DNS 三者的關係。
  • 聽到「網站打不開」時,知道這可能跟「找不到地方」有關,而不只是「Server 壞了」。
情境

「我們才剛換新網域,結果客戶說打不開」

Northstar Electronics 上週把官網從舊網址搬到新網域,IT 同仁收到反映:

「有些客戶說新網址打不開,但我們同事這邊都正常,是不是伺服器有問題?」

這個情境很典型:換網域之後,網址要花一點時間才會被全世界「認得」。這跟 Server 本身好不好一點關係都沒有,而是跟這一章要教的 DNS 有關。搞懂這個機制,您就能理解為什麼工程師常說「再等等,DNS 還在生效中」。

情境例子 1/2

門牌地址,還是背後真正的座標

想像您要寄信給一位朋友,信封上寫的是「幸福路 100 號」,郵差看得懂這個地址,就能把信送到。但郵差心裡真正認的,其實是這棟房子在地圖上精確的座標位置,「幸福路 100 號」只是給人類方便記憶的門牌。

Domain(網域名稱)就像那個門牌地址,例如 northstar-electronics.com,是給人類方便記憶、輸入的名字。

IP 位址則是電腦世界裡真正的座標,像 203.0.113.25 這樣一串數字。Server 真正的身分,靠的是這串數字,不是好記的名字。

先記住這句 Domain 是給人看的門牌,IP 是機器認得的座標。兩者指向同一個地方,只是給不同「讀者」使用。
情境例子 2/2

那 DNS,就是查地址的電話簿

問題來了:如果 Server 只認 IP,那使用者打的 Domain 要怎麼變成 IP?這裡需要一位「翻譯員」。

DNS(Domain Name System,網域名稱系統)就是這個翻譯員,它像一本全世界共用的電話簿:您輸入姓名(Domain),它幫您查出對應的電話號碼(IP),然後您的 Browser 才會拿著這個 IP,去敲 Server 的門。

回到開頭的情境:Northstar 換了新網域,等於在電話簿裡登記了一筆新資料。這本「電話簿」分散在全世界很多台電腦上,更新需要時間慢慢傳播出去,這段等待期,就是工程師說的「DNS 生效中」。有些人的電腦已經查到新資料,有些人的電腦還拿著舊資料,才會出現「同事這邊正常、客戶那邊打不開」的落差。

正式定義

拆解一個完整的網址

先看正式定義,再拆解網址的每一段:

名詞定義
IP 位址電腦在網路上的數字位址,是機器真正用來定位彼此的座標
Domain
網域名稱		給人類方便記憶、輸入的網址名稱,對應到某個 IP 位址
DNS把 Domain 翻譯成對應 IP 位址的系統
URL完整的網址,包含通訊協定、Domain、路徑等資訊

用一個實際的網址拆解:https://www.northstar-electronics.com/products

  • https:// — 通訊協定(單元 2 的下一章會深入教 HTTP/HTTPS)
  • www.northstar-electronics.com — Domain,DNS 要查的就是這一段
  • /products — 路徑,告訴 Server 您想看網站裡的哪個頁面
串起來看

從打網址到看見畫面的完整旅程

回顧 1-2 教過的一問一答,現在補上 DNS 這一段:

1. 您在 Browser 輸入 www.northstar-electronics.com
2. Browser 問 DNS:「這個 Domain 對應的 IP 是什麼?」
3. DNS 回答:「是 203.0.113.25」
4. Browser 拿著這個 IP,向真正的 Server 發出 Request
5. Server 回傳 Response,Browser 把內容畫成畫面

第 2、3 步是這一章新補上的環節,發生在第 4 步(1-2 教過的 Client 向 Server 發請求)之前。少了這一段翻譯,Browser 根本不知道該敲哪扇門。

為什麼要學這個

兩個馬上用得到的場景

場景一:判斷「網站打不開」是哪一種情況

換網域或搬遷主機後,客戶反映「打不開」,先確認是不是 DNS 生效還沒傳播完成,而不是急著往 Server 本身找問題。這種情況通常過一段時間(數小時到一兩天)會自行恢復,不需要工程師緊急介入。

場景二:向客戶解釋為什麼換網域需要「觀察期」

了解 DNS 的運作,您就能提前告訴客戶:「網域切換後,可能有一小段時間內,不同地區的使用者看到的網站會不一樣,這是正常現象,不是系統故障。」提早說明,能避免上線後不必要的緊張回報。

核心概念 1/2

Domain 和 IP,是兩種不同的「認得」

概念一:Domain 方便人類,IP 才是機器的真身分

人類記名字比記數字容易,所以有了 Domain 這層方便的介面。但別忘了,Server 真正的身分認證,底層還是靠 IP 位址在運作。

概念二:一個 Domain 可以對應到不同的 IP

企業可能因為維護、切換機房等原因,把同一個 Domain 指向新的 IP。這時候 DNS 裡的紀錄要更新,而更新傳播到全世界需要時間,這就是「DNS 生效中」這句話的由來。

核心概念 2/2

DNS 是分散式的電話簿,不是單一個檔案櫃

概念三:DNS 查詢結果會被暫存

為了不用每次都查一輪電話簿,使用者的裝置和沿路的網路設備,會把查到的結果暫存一段時間。這也是為什麼即使 DNS 紀錄已經更新,有些人還是會看到舊的內容一段時間——他們的裝置還拿著暫存的舊答案。

概念四:DNS 出問題,影響的是「找路」,不是「內容本身」

如果 DNS 查詢失敗,Browser 根本連 Server 的門都敲不到,連 Request 都發不出去。這跟 Server 本身處理請求時發生錯誤,是完全不同層級的問題,排查方向也不一樣。

容易混淆

兩組容易搞混的概念

容易混淆的一組怎麼分
Domain vs URLDomain 只是網址中的「名稱」那一段;URL 是包含通訊協定、Domain、路徑在內的完整網址。Domain 是 URL 的其中一部分。
「網站打不開」的兩種可能一種是 DNS 查不到對應的 IP(連門都找不到);另一種是 IP 找到了,但 Server 本身沒有正常回應(門找到了,但沒人應門)。兩種情況的排查方向完全不同。
案例

Northstar Electronics 的網域切換

Northstar 把官網從舊網域 old-northstar.com 切換到新網域 northstar-electronics.com:

切換當天:IT 團隊在 DNS 系統裡,把新網域指向正式環境的伺服器 IP
接下來 24 小時:世界各地的使用者,陸續開始看到新網址正常運作
少數使用者:因為自己裝置暫存了舊的 DNS 查詢結果,還要多等一些時間才會更新
  1. 有哪些角色?IT 團隊、DNS 系統、世界各地的使用者裝置。
  2. 資料如何移動?DNS 紀錄的更新,需要時間傳播到全世界的 DNS 伺服器與使用者裝置的暫存。
  3. 各角色負責什麼?IT 團隊負責在 DNS 系統裡設定正確的對應;傳播與暫存則是 DNS 機制本身的運作方式,不是人為可以立刻控制的。
  4. 哪裡可能出錯?如果客戶反映「打不開」就急著檢查 Server,反而可能白忙一場,因為 Server 本身根本沒問題,問題卡在 DNS 傳播還沒完成。
  5. 第一個合理的檢查方向?先確認距離切換的時間有多久,再確認是不是只有少數使用者受影響——這兩個線索能幫助判斷是不是單純的 DNS 傳播延遲。

常見誤解

三個很多人踩過的坑

誤解一:「網址打不開,一定是 Server 掛了」

也可能是 DNS 還沒查到正確的 IP,Request 根本還沒送到 Server。先確認是不是最近才換過網域或伺服器位置。

誤解二:「DNS 更新後,全世界立刻都會看到新內容」

DNS 的更新需要時間傳播,加上各地裝置的暫存,不同使用者看到新內容的時間點可能相差數小時甚至更久。

誤解三:「Domain 就是 Server 本身」

Domain 只是一個好記的名字,指向某個 IP。同一個 Server 可以有好幾個 Domain 指向它,也可能因為搬遷,同一個 Domain 改指向不同的 Server。

遇到問題先查什麼

「網站打不開」的初步檢查順序

  1. 確認是否最近變更過網域或伺服器設定。如果有,先考慮 DNS 傳播延遲的可能性。
  2. 確認影響範圍。只有部分使用者,還是所有人都打不開?部分使用者受影響,更像是 DNS 暫存的落差。
  3. 換一個網路環境測試。換用手機網路或其他電腦,看看是否能正常開啟。
  4. 確認距離變更已經過了多久時間。DNS 傳播通常需要數小時到一兩天。
  5. 如果時間已經足夠久仍然打不開,再往 Server 本身的問題排查。這時候才需要請工程師確認伺服器狀態。
重點整理

帶走這四句話

  1. Domain 是給人類方便記憶的名字,IP 才是機器真正認得的座標。
  2. DNS 是把 Domain 翻譯成 IP 的系統,像一本全世界共用的電話簿。
  3. DNS 的更新需要時間傳播,加上裝置的暫存,不是切換後立刻全世界同步生效。
  4. 「網站打不開」有兩種完全不同的可能:找不到路(DNS),或找到了但沒人應門(Server)。

檢核 2/4

是非題:網域切換後,全世界的使用者會立刻同時看到新的內容。

正確答案是「否」。DNS 紀錄的更新需要時間傳播到全世界,加上使用者裝置本身可能暫存了舊的查詢結果,不同人看到新內容的時間點會有落差。
檢核 3/4

下列哪一項才是機器真正用來定位 Server 的資訊?

正確答案是 C。Domain 是給人類方便記憶的名字,機器真正用來定位彼此的,是背後對應的 IP 位址。
檢核 4/4

情境題:企業剛換了新網域,少數客戶反映打不開,多數客戶正常。第一個合理的懷疑方向是?

正確答案是 A。多數人正常、少數人異常,而且剛換過網域,最可能的原因是 DNS 傳播延遲或裝置暫存了舊紀錄,不是 Server 本身故障。
