HTML 如何構成網頁
Tag、Element 與 Attribute

預估 30 分鐘 ・ 建議先讀完 1-1 再來。

單元 1、2 教了系統架構和網路運作,這個單元要正式進入前端的世界,從最基礎的 HTML 開始。您不需要學會寫程式,只需要看懂結構,能對照著檢查問題。

  • 說明 HTML 在網頁裡扮演的角色。
  • 認得 Tag、Element、Attribute 這三個名詞。
  • 看到一小段 HTML 原始碼時,能大致猜出這段畫面長什麼樣子。
圖解

HTML 是一棵有父子關係的樹

情境

「工程師說欄位少了一個 Attribute,我聽不懂」

Northstar Electronics 的官網有個報名表單,顧問向工程師反映某個欄位沒有正確顯示提示文字,工程師回覆:

「喔,那應該是那個 input 少了 placeholder 這個 Attribute,補上就好了。」

如果完全不懂 HTML,這句話聽起來像另一種語言。但只要理解 1-1 教過的「前端負責呈現畫面」,再加上這一章的三個基本名詞,您就能聽懂工程師在講的,其實是一件很具體、很小的事。

情境例子 1/2

樂高積木和積木說明書

想像網頁是一座用樂高積木堆出來的建築。每一塊積木都有自己的形狀和用途:有的是牆壁、有的是窗戶、有的是屋頂。HTML(HyperText Markup Language,超文本標記語言)就是那份告訴瀏覽器「這裡放什麼形狀的積木」的說明書。

每一塊積木的種類,叫做 Tag(標籤),例如 <p> 代表「這裡放一段文字」,<img> 代表「這裡放一張圖片」。而一塊已經放上去、實際存在畫面裡的積木,叫做 Element(元素)——Tag 是積木的「種類名稱」,Element 是「放進畫面裡的那一塊實體」。

先記住這句 Tag 是說明書上寫的種類,Element 是實際放進畫面的那一塊。日常對話裡兩者常被交替使用,但正式定義上,Element 才是真正出現在網頁裡的東西。
情境例子 2/2

積木上的貼紙說明

有些積木上,還會貼一張小貼紙,寫著額外的說明:「這塊牆是紅色的」「這扇窗戶要開在這個位置」。這些額外的說明,就是 Attribute(屬性)——附加在 Element 上,提供更多細節的資訊。

回到開頭工程師說的那句話。input 是一種 Tag(用來放輸入欄位這塊積木);placeholder 就是貼在這塊積木上的一張貼紙,寫著「欄位裡要先淡淡顯示的提示文字是什麼」。少了這張貼紙,積木本身還在,只是少了提示文字這個細節,不影響欄位能不能正常輸入,但體驗上少了引導。

正式定義

三個名詞的正式說法

名詞定義例子
Tag
標籤		HTML 裡用尖括號包起來的標記,定義這是哪一種內容<p>、<img>
Element
元素		由 Tag(通常是開始和結束一對)加上裡面內容,共同組成的一個實際單位<p>您好</p>
Attribute
屬性		寫在 Tag 裡面,提供額外設定或資訊的項目placeholder="請輸入姓名"

看一個實際範例:<input type="text" placeholder="請輸入姓名">——input 是 Tag,整段合起來(含屬性)是一個 Element,type 和 placeholder 都是 Attribute,分別說明「這是文字輸入欄位」和「提示文字是什麼」。

串起來看

一小段 HTML,對照畫面長什麼樣子

看一段簡短的 HTML,逐行對照它在畫面上會呈現的樣子:

<div>
  <p>歡迎報名 Northstar 年度活動</p>
  <img src="banner.jpg" alt="活動banner">
  <a href="/register">立即報名</a>
</div>

逐行解釋:第一行 div 是一個容器,把裡面幾樣東西包在一起(3-2 會深入教);第二行 p 顯示一段文字;第三行 img 顯示一張圖片,src 這個 Attribute 告訴瀏覽器圖片檔案在哪裡;第四行 a 顯示一個可以點擊的連結,href 這個 Attribute 告訴瀏覽器點下去要跳去哪個網址。短短四行,就能對應出畫面上「一段文字、一張圖片、一個按鈕」的樣子。

為什麼要學這個

兩個馬上用得到的場景

場景一:聽懂工程師描述畫面問題的用語

工程師說「這個 Element 少了某個 Attribute」,您現在能理解這是在講一塊具體的畫面積木,缺了某個設定細節,而不是天書。這能讓您在會議中跟上討論,甚至幫忙補充觀察到的畫面現象。

場景二:自己簡單檢查畫面元素

結合 2-8 教過的開發者工具,您可以用「檢查」功能點選畫面上的某個區塊,直接看到它對應的 HTML 內容,確認是不是少了某個該有的文字或連結設定。

核心概念 1/2

HTML 只負責「結構」,不負責「好不好看」

概念一:HTML 描述的是「這裡有什麼」,不是「長什麼樣子」

HTML 只負責定義畫面上有一段文字、一張圖片、一個按鈕,並不負責顏色、大小、間距這些外觀細節。外觀是 3-4 會教的 CSS 負責的工作,兩者是分工合作的關係。

概念二:大多數 Tag 都有開始和結束一對

像 <p>...</p> 這樣成對出現,中間夾的內容就是這個 Element 的實際內容。少數 Tag(像 <img>)不需要成對的結束標籤,因為它本身不包裹任何文字內容。

核心概念 2/2

Attribute 是細節,但細節有時很重要

概念三:同一種 Tag,可以透過不同 Attribute 產生完全不同的行為

同樣都是 input 這個 Tag,type="text" 是一般文字欄位,type="checkbox" 就變成核取方塊,行為完全不同。Attribute 雖然是「細節」,但很多時候決定了這個 Element 實際上是什麼。

概念四:少了某個 Attribute,通常不會讓畫面「壞掉」,但可能少了某個功能

就像開頭的例子,少了 placeholder,欄位還是能正常輸入,只是少了提示文字的體驗。這也是為什麼有些「小問題」看似無傷大雅,但顧問若能敏銳地發現,能提升產品的完整度。

容易混淆

兩組容易搞混的概念

容易混淆的一組怎麼分
Tag vs ElementTag 是標記本身(例如 <p>);Element 是這個標記加上內容後,構成的完整單位(例如 <p>您好</p>)。日常對話中常混用,不用太糾結,但正式書寫時可以留意這個差異。
HTML vs CSSHTML 定義畫面上「有什麼內容」(結構);CSS(3-4 會教)決定這些內容「長什麼樣子」(外觀)。兩者分工不同,常常被新手搞混成同一件事。
案例

Northstar Electronics 的活動報名頁

Northstar 的活動報名頁面,顧問發現圖片旁邊少了替代文字,截圖詢問工程師,對方查看後回覆:

原本的 HTML:<img src="banner.jpg">
建議修改:<img src="banner.jpg" alt="2026年度活動主視覺">
  1. 有哪些角色?顧問(發現問題的人)、工程師(修改 HTML 的人)。
  2. 資料如何移動?這裡沒有涉及系統間的資料流動,單純是這個頁面的 HTML 結構本身需要補上一個 Attribute。
  3. 這個 Attribute 負責什麼?alt 屬性用來提供圖片的替代文字,在圖片載入失敗,或給視障使用者的輔助工具閱讀時,會顯示這段說明。
  4. 哪裡可能出錯?如果沒有這個屬性,圖片一旦載入失敗,使用者只會看到一片空白,不知道原本該是什麼內容。
  5. 第一個合理的檢查方向?用 2-8 教過的開發者工具「檢查」功能,直接點選這張圖片,查看它的 HTML 內容裡有沒有帶上完整的說明性屬性。

常見誤解

三個很多人踩過的坑

誤解一:「HTML 就是網頁設計,決定了畫面好不好看」

HTML 只負責定義內容結構,外觀的美感是 CSS 的工作。把兩者混為一談,容易誤會問題出在錯誤的地方。

誤解二:「Attribute 都是無關緊要的細節」

有些 Attribute(像剛才提到的 type)會直接決定 Element 的行為,不是單純的裝飾性資訊,不能一概而論地忽略。

誤解三:「只要看得懂英文字,就等於看得懂 HTML」

HTML 有自己的結構規則(開始標籤、結束標籤、巢狀關係),不是單純認得單字就能完全理解,但作為顧問,理解基本結構的概念就已經足夠應付大多數溝通情境。

遇到問題先查什麼

「畫面內容有問題」的檢查順序

  1. 用開發者工具的「檢查」功能,點選出問題的畫面區塊。能直接看到對應的 HTML 內容。
  2. 確認這是哪一種 Tag。是文字、圖片,還是連結?
  3. 檢查這個 Element 有沒有帶上該有的 Attribute。例如圖片有沒有 src、連結有沒有 href。
  4. 確認內容本身是否正確。是結構的問題,還是內容打錯了。
  5. 把觀察到的具體 HTML 內容提供給工程師。比只描述「畫面怪怪的」更精準。
重點整理

帶走這四句話

  1. HTML 是描述網頁結構的標記語言,決定畫面上「有什麼內容」。
  2. Tag 是標記的種類,Element 是實際放進畫面的那個單位,Attribute 是附加在上面的細節說明。
  3. HTML 只負責結構,不負責外觀,外觀是 CSS 的工作。
  4. Attribute 雖是細節,但有時直接決定了 Element 的行為,不能一概當作無關緊要。
檢核 1/4

HTML 主要負責網頁的哪個部分?

正確答案是 A。HTML 負責定義網頁的內容結構,外觀樣式是 CSS 負責的工作。
檢核 2/4

是非題:Attribute 只是裝飾性資訊,不會影響 Element 的實際行為。

正確答案是「否」。例如 input 的 type 屬性,會直接決定這是文字欄位還是核取方塊,並非只是裝飾用途。
檢核 3/4

在 <img src="banner.jpg" alt="活動圖片"> 這段 HTML 裡,alt="活動圖片" 是什麼?

正確答案是 C。alt 是附加在 img 這個 Element 上的屬性,提供圖片的替代文字說明。
檢核 4/4

情境題:客戶反映某張圖片載入失敗時,畫面上什麼提示都沒有。最可能缺少的是什麼?

正確答案是 A。少了 alt 屬性,圖片載入失敗時就不會顯示任何替代文字說明,這是很常見、也很容易被忽略的小細節。
